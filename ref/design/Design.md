# KEESS Design.md — B안 디자인 헌법 (Design Constitution · B-Type)

> 이 문서는 KEESS **B안** 사이트의 최상위 디자인 기준이다. 수백 개 화면을 만들거나 Claude가 코드를 생성·수정하더라도, 모든 페이지가 **하나의 사이트**로 보이도록 강제하는 규칙 문서다.
>
> **본 문서는 신규 디자인을 정의하지 않는다.** 확정 홈(`keess_home_C_v18_최종확정`)을 최상위 기준으로 하고, B안 확정 프로토타입에서 공통으로 검증된 디자인 언어만 추출·체계화했다. A안과 **동일한 디자인 시스템**을 공유하며(토큰·타이포·모션·컴포넌트 1:1), B안 특화 기능(정부지원 환급·과정리스트 다운로드·크로스링크)과 아래 **§0.5 B안 필수 준수 6대 원칙**을 추가로 담는다.
> 모든 규칙은 **WHY(왜) · WHEN(언제) · HOW(어떻게)** 형태로 기술한다.
>
> B안 확정 기준 파일(source of truth):
> `keess_home_C_v18_최종확정(260703).html`(홈 · A/B 공통), `keess_C_corporate_v3_확정.html`(코퍼레이트), `keess_P1_AXAI_D_scenario_v7.html`(P1 · A/B 공통), `keess_P2_leadership_B_framework_v1.0.html`(P2), `keess_P3_hrd_B_platform_v2.0.html`(P3), `keess_P4_content_solution_B_v2.0.html`(P4).

## 0. 사용 규칙 (이 문서를 읽는 Claude에게)
- **WHY**: 일관성은 신뢰다. B2B/B2G 구매 결정권자는 "정돈된 사이트 = 정돈된 파트너"로 읽는다.
- **WHEN**: 모든 페이지·컴포넌트·카피 생성 및 수정 전에 이 문서를 먼저 참조한다.
- **HOW**:
  - 새 값(컬러/폰트/그림자/간격/컴포넌트)을 발명하지 않는다. 아래 토큰·컴포넌트만 조합한다.
  - 필요한 패턴이 여기 없으면 → 임의 생성 금지. 홈/P1~P4 확정 파일에서 동일 목적의 기존 패턴을 찾아 재사용한다.
  - 규칙 충돌 시 우선순위: **홈 확정본 > 코퍼레이트 확정본 > P1~P4 확정 프로토타입**.
  - **A안과 B안은 하나의 Design.md를 공유한다.** 본 문서는 그 위에 B안 특화 규칙만 얹은 것으로, 시각 시스템(§3·§4·§6)은 A안과 완전히 동일하다.

## 0.5 B안 필수 준수 6대 원칙 (★ 위반 금지 · A안 이슈 재발 방지)

A안 빌드에서 나온 문제를 B안에서 반드시 방지하기 위한 최우선 규칙. 아래 6개는 다른 어떤 세부 규칙보다 우선한다.

1. **GNB 네비게이션 = 실제 페이지 라우팅**
   - **WHY**: 상단 메뉴가 클릭되지 않거나 앵커로만 동작하면 사이트가 "죽은" 느낌을 준다.
   - **HOW**: GNB의 각 항목(AX·AI 전환 / 리더십·조직 / HRD 통합 솔루션 / 콘텐츠 솔루션 / 정부지원)을 클릭하면 **각각의 실제 페이지로 이동**한다(`/ax-ai` `/leadership` `/hrd` `/content`, 정부지원은 `/hrd#gov`). 현재 페이지 항목은 `aria-current="page"`로 활성 표기. 무동작(no-op)·placeholder 링크 금지.

2. **좌측 상단 KEESS 로고 = 홈으로 이동**
   - **WHY**: 로고→홈은 웹 표준 기대 동작. 없으면 방문자가 길을 잃는다.
   - **HOW**: 좌측 상단 `KG KEESS` 로고 클릭 시 **항상 홈(`/`)으로 랜딩**. 홈에서는 최상단 스크롤로 처리 가능.

3. **그라데이션 = 원래 있던 영역이면, 디자인 톤 준수하에 적용 가능**
   - **WHY**: A안은 그라데이션을 전부 제거(플랫)했으나, 확정 프로토타입이 원래 그라데이션을 쓰던 영역(히어로 배경·미디어 스크림·CTA 강조 등)은 **톤을 지키면 그라데이션이 오히려 완성도를 높인다.**
   - **HOW**: **확정 프로토타입에서 이미 그라데이션이 적용돼 있던 영역에 한해** 그라데이션 허용. 단 (a) 색은 해당 필러 포인트색(`--p1~--p4`)·중립(`--ink`)에서 파생, (b) 무지개·이질적 다색 혼합 금지, (c) 대형 본문 텍스트 배경 채색 금지. → **"새 그라데이션 발명"은 금지, "원래 있던 그라데이션의 톤 정합 재현"은 허용.**

4. **인터랙션 고도화 허용 (디자인 톤 안에서)**
   - **WHY**: 확정 구성·카피는 유지하되, 체감 품질을 높이는 미세 인터랙션은 사이트를 "비싸" 보이게 한다.
   - **HOW**: 작업 중 **더 자연스럽게 만들 수 있는 인터랙션(hover 피드백·포커스 상태·부드러운 전환·스크롤 스파이·스켈레톤 등)이 있으면 고도화 가능.** 단 §6 모션 값·이징 토큰 안에서, 구성(IA·섹션 순서·카피)은 변경하지 않고, `prefers-reduced-motion` 대응 유지. 바운스·무한 반복·1s 초과 등장은 여전히 금지.

5. **푸터 = 전 페이지 공통 (홈 다크 푸터)**
   - **WHY**: 푸터는 사이트의 마침표. 페이지마다 다르면 통일감이 깨진다.
   - **HOW**: **홈 확정 다크 푸터(`.foot`)를 전 페이지에서 그대로 재사용.** 필러 네비·회사정보·SNS·패밀리 메뉴 + **부정훈련 예방/신고**(§5-B, A/B 공통 고정) 포함. 페이지별 미니 푸터·와이어프레임 푸터 금지.

6. **'교육 상담' CTA = 최상의 UI/UX로 "누르고 싶게"**
   - **WHY**: 교육 상담은 KEESS의 **유일한 전환 지점**. 이 버튼의 매력이 곧 리드 수다.
   - **HOW**: 상단 고정 nav의 "교육 상담" CTA와 각 페이지 최종 CTA는 **가장 자연스럽고 누르고 싶은** 형태로 다듬는다:
     - 고대비 `btn-ink`(또는 필러 강조) + pill + 아이콘(화살표/말풍선), hover `translateY(-2px)`+shadow 상승, active 미세 눌림.
     - 원래 그라데이션이 있던 CTA라면 원칙 3에 따라 톤 정합 그라데이션 허용(강조 상승).
     - 마이크로카피는 동사형·이득 중심("교육 상담 신청", "무료로 상담받기" 등 확정 카피 우선). 과장·이모지 금지.
     - 항상 **1차 CTA는 하나**, 최종 행동은 상담/진단/가이드/다운로드로 수렴. 클릭 영역 충분(터치 44px+), 포커스 링 명확.

## 1. Project Philosophy (프로젝트 철학)

**WHY — KEESS는 무엇인가**
- KEESS는 KG에듀원 HRD사업본부의 **기업·기관 교육 도입 문의 채널(B2B·B2G 인바운드)**이다. 결제·가격 노출이 없는, 제품 소개와 상담 전환 중심 플랫폼이다.
- KEESS는 "교육을 파는 곳"이 아니라 **"조직 전환(Transformation)을 설계하는 파트너"**로 스스로를 규정한다.

**핵심 명제 (사이트 전체를 지배하는 한 문장)**
> "진단으로 설계하고, 효과로 증명합니다." — 확정 홈 히어로

- 모든 화면은 이 명제의 변주다: **먼저 진단 → 직무로 좁혀 설계 → 실행 → 숫자로 증명.**
- "과정을 고르기 전에, 조직을 먼저 진단합니다."(홈)가 서비스 철학의 표준 표현이다.

**WHEN/HOW**
- 어떤 페이지든, 방문자가 "이 회사는 우리 조직을 이해하고 성과를 책임진다"고 느끼게 한다.
- 기능·커리큘럼 나열보다 **문제 정의 → 해결 여정 → 증거**의 서사를 우선한다.

## 3. Visual Identity (비주얼 아이덴티티) — A안과 동일

### 3-1. Color Tokens (확정 홈 기준, 절대 신규 생성 금지)
```css
/* Neutral base — 홈/코퍼레이트 canonical */
--bg:#FAFAFB;      /* 페이지 배경 */
--ink:#14141A;     /* 기본 텍스트·다크 surface·버튼 */
--muted:#54585f;   /* 보조 텍스트·eyebrow */
--line:#E6E8EC;    /* 구분선·테두리 */
--surface:#F3F5F8; /* 카드·톤 박스 배경 */

/* Pillar accents — 포인트 컬러 전용 */
--p1:#2E1A6B; /* AX·AI 전환 · Royal Purple */
--p2:#E91E63; /* 리더십·조직 · Magenta */
--p3:#8B27A8; /* HRD 통합 솔루션 · Violet */
--p4:#F58220; /* 콘텐츠 솔루션 · Orange */
--gov:#F4B83A;/* 정부지원 · Yellow (크로스 배지) */
```

**Pillar 톤 규칙 (P1~P4 페이지에서 검증됨)**
- **WHY**: 페이지마다 정체성은 주되, 사이트는 하나로 보여야 한다.
- **HOW**: 필러 페이지는 중립 베이스를 **각 필러색으로 아주 옅게 틴트**한다. 필러색 자체(#2E1A6B 등)는 **포인트(배지·번호·강조·CTA 강조)에만** 쓴다. 페이지 전체를 필러색으로 칠하지 않는다.
  - P1(AX·AI): `--bg:#FCFCFE --surface:#F4F2FB --ink:#15131D --line:#EAE8F2` (보라 틴트)
  - P2(리더십): `--bg:#FDFBFC --surface:#FCEBF2 --ink:#181318 --line:#F0E3EA` (마젠타 틴트)
  - P3(HRD): `--surface:#F4EAFA --line:#ECE0F4 --ink:#1A1420` (바이올렛 틴트)
  - P4(콘텐츠): `--surface:#F7F4EF --ink:#15131D` (웜 틴트)
- **홈/코퍼레이트**는 틴트 없는 중립 베이스(#FAFAFB) + 4색을 필러 구분에 병렬 사용.
- **그라데이션(원칙 3 연동)**: 확정 프로토타입에서 이미 그라데이션이 있던 영역(히어로 배경·미디어 스크림·강조 CTA·데이터 시각화 바/노드)은 **해당 필러색·중립에서 파생한 톤 정합 그라데이션**을 허용. 신규 다색 그라데이션 발명·대형 텍스트 배경 채색은 금지.
- **DON'T**: 필러색을 큰 배경면·대형 텍스트 전체에 사용. 4색을 한 컴포넌트에 동시 혼합.

### 3-2. Typography
```css
--serif:"Gowun Batang",serif;                 /* 디스플레이/선언(manifesto) 전용 */
/* UI·본문 */ font-family:"Pretendard Variable",Pretendard,system-ui,sans-serif;
body: line-height:1.6; letter-spacing:-.01em; -webkit-font-smoothing:antialiased;
```
- **Pretendard Variable**(self-host, `next/font/local`, weight `45 920`, `--font-pretendard`) = 기본 UI·본문·헤딩. 전 페이지 통일. **Gowun Batang(serif)** = 선언형 대형 문구(매니페스토 등)에만 제한적으로.
- **원본 폰트 불일치 정규화(B안 필수)**: 확정 프로토타입의 P2·P3는 `"Pretendard","맑은 고딕"` 폴백 + 웹폰트 미로드로 머신별 렌더가 다르다. **빌드 시 5페이지 전부 self-host Pretendard Variable 하나로 통일**한다. CDN 링크·Malgun 폴백 스택을 그대로 옮기지 않는다.
- **Type scale (clamp, 반응형) — 검증된 값만 사용**:
  - Hero H1: `clamp(40px,6.8vw,84px)` 또는 `clamp(34px,6vw,74px)`
  - Manifesto(serif): `clamp(30px,5.2vw,60px)`, line-height 1.25
  - Section title: `clamp(27px,3.8vw,42px)` / `clamp(26px,4vw,46px)`
  - Sub/Lead: `clamp(16px,1.8vw,20px)` / `clamp(15px,1.7vw,19px)`, line-height 1.7, max-width ~54ch
  - Body: 14~15px, line-height 1.6~1.7
  - **Eyebrow**: 12.5px, `letter-spacing:.16em`, `text-transform:uppercase`, weight 700, color `--muted`
- **WHY 폰트 2종만**: 다중 서체는 통일성을 깬다. 확장 금지.

### 3-3. Spacing / Grid / Layout tokens
```css
--maxw:1200px; /* 홈·코퍼레이트 */   /* 필러 페이지: 1180px */
--gut:24px;    /* 좌우 거터 */
```
- 섹션 상하 여백: `.section` 120px(desktop)/74px(mobile); `.pillar` 108px/70px. **섹션 간 리듬을 이 값으로 유지.**
- 콘텐츠 최대폭: 홈 1200 / 필러 1180. 본문 텍스트 가독폭 ~54ch.

### 3-4. Radius / Elevation / Surface
```css
--r:20px;
--shadow-1:0 1px 2px rgba(20,20,26,.05),0 2px 8px rgba(20,20,26,.05);
--shadow-2:0 2px 6px rgba(20,20,26,.06),0 10px 26px rgba(20,20,26,.09);
--shadow-3:0 6px 14px rgba(20,20,26,.08),0 22px 52px rgba(20,20,26,.14);
```
- Radius 규칙: 카드/surface = `--r`(20px); 버튼·칩·필 = `999px`(pill); 모달(pv-dialog) = 22px; 소형 메뉴·드롭다운 = 12px.
- Elevation은 위 3단계만. 임의 그림자 금지.

### 3-5. Icon / Illustration / Image
- 아이콘: 인라인 SVG, 얇은 라인·모노톤(텍스트색 상속). 컬러 아이콘 남발 금지.
- 이미지: 히어로/필러 미디어는 대형 배경 이미지 + 스크림(scrim) 오버레이로 텍스트 대비 확보. Ken Burns 류 느린 확대만 허용.
- 일러스트: 확정본에 없는 장식 일러스트 신규 도입 금지. 데이터·구조 도식 위주.

## 4. Layout System — A안과 동일

- **Container**: `.wrap`류, `max-width:var(--maxw)`, 좌우 `--gut`.
- **Section**: 세로 리듬 120/74(pillar 108/70). 섹션 구분은 `border-top:1px solid var(--line)` 또는 배경 톤 전환.
- **Hero 구조 (표준)**: 다크/이미지 배경 → `eyebrow` → `h1`(대형, 단어별 등장) → `sub` → `actions`(CTA 1차+2차) → 하단 `chips`/KPI. 홈 히어로는 5장 캐러셀.
- **CTA 구조**: 1차 = `btn-ink`(고대비), 2차 = `btn-glass`/`btn-line-dark`. 항상 **1차는 하나**. 최종 CTA는 상담/진단/가이드/다운로드로 수렴. (원칙 6 연동)
- **Sticky Navigation**: `.nav` fixed, `z-index:70`, height 72px. 스크롤 시 배경·보더 페이드 전환. **GNB=실제 라우팅(원칙 1), 로고=홈(원칙 2), 우측 "교육 상담" CTA 고정(원칙 6).** 모바일은 `.mmenu` 풀스크린 오버레이. 페이지별 서브내비(스크롤 스파이) 허용.
- **Footer**: `.foot` 다크(`--ink`) 배경. **전 페이지 공통(원칙 5).** 부정훈련 예방/신고 포함.
- **Breakpoints (검증된 값만)**: 1040 / 940 / 880 / 820 / 760 / 740 / 720 / 640 / 560px. 신규 임의 브레이크포인트 금지. `@media(prefers-reduced-motion:reduce)` 대응 필수.
- **우측 도구**: 페이지 도트(`.pdots`), 최상단 이동 FAB(`.to-top`).

## 5. Component Principles — A안과 동일 (+B안 nav/CTA 강화)

- **Nav (`.nav`/`.nav-in`)** — WHY: 상시 전환 경로 유지. HOW: fixed 72px, 스크롤 배경 전환. **GNB 각 항목 = 실제 페이지 링크(원칙 1), 로고 = 홈(원칙 2), 우측 "교육 상담" CTA 고정(원칙 6).** 현재 페이지 `aria-current`. 금지: 무동작 링크, 스크롤 시 사라지는 nav, 메뉴 과다.
- **Section Header (eyebrow + title + sub)** — 영문 eyebrow(대문자·자간 .16em) → 국문 title(clamp) → sub 리드. 금지: eyebrow 없이 본문 시작.
- **Metric / KPI (`.stat`/`.kpi`/`.val`)** — 큰 숫자 + 짧은 라벨, 카운트업 허용. 금지: 근거 없는 수치.

- **Button (`.btn` 계열)** — `btn-ink`(1차), `btn-glass`(히어로 위), `btn-line-dark`(보조). 모두 pill, hover `translateY(-2px)`+shadow. **"교육 상담" 등 핵심 CTA는 원칙 6에 따라 최상의 체감으로 강화(톤 정합 그라데이션 허용).** 금지: 한 화면 1차 버튼 다수.

- **Tab / Accordion (`.faq-tab`/`.faq-item`)** — 아웃라인 pill 탭, 셰브론 회전 + 높이 애니메이션. 금지: 기본 전체 펼침.

- **Carousel (`.hero-slide`/`.hs-*`)** — 홈 히어로 5장, Ken Burns, 도트, 자동+수동. 금지: 상세 히어로 캐러셀 남용.
- **Floating CTA / To-top (`.to-top`)** — 우하단 48px FAB. 금지: 콘텐츠 가리는 대형 부유 배너. **(B안: 과정리스트 다운로드는 플로팅 금지 — §5-B 참조)**
- **Contact / Diagnostic Form (`.field`/`.req`/`.err`)** — 회사·담당자·이메일·규모·관심영역, 필수 표시·인라인 검증·개인정보 동의, B2B/B2G 부문 선택. 금지: 결제·가격 필드.

## 6. Motion System — A안과 동일 (+원칙 4 고도화 허용)

```css
--ease:cubic-bezier(.22,.61,.36,1);
--ease-out:cubic-bezier(.2,0,0,1);
```
- **Scroll Reveal (표준)**: `.r{opacity:0;transform:translateY(24px)}` → `.in`. `IntersectionObserver({threshold:.16})` **통일**(원본 P3/P4의 .12는 .16으로 정규화), 1회 unobserve, transition `.7s var(--ease)`.

- **Timing 표준**: 등장 .7~.9s, 인터랙션 .2~.35s, 미세 .1~.18s.
- **인터랙션 고도화(원칙 4)**: 위 토큰·타이밍 안에서 hover/focus/전환/스크롤 스파이/스켈레톤 등 체감 품질 향상 허용. 구성·카피 불변, reduced-motion 유지.

- **DON'T**: 바운스·과한 스프링, 무한 반복 강조, 1s 초과 등장.

## 11. Do / Don't

**DO**

- 한 화면 1차 CTA 다수, 전환 경로 분산. **과정리스트 플로팅 다운로드 신설.**
- 가격·결제·장바구니 요소 추가. 바운스/무한 반복/1s 초과 모션. 확정 구조 임의 변경.

## 12. Coding Convention

- **스택**: Next.js(App Router) + React + TypeScript + Tailwind v3.4. Pretendard self-host(`next/font/local`), Gowun Batang(`next/font/google`, 홈 한정).

- **JS**: 스크롤 리빌 공통 `IntersectionObserver(threshold:.16)` 재사용. 초기화는 셀렉터 기반, 전역 오염 최소화.

## 13. Future Rule (새 페이지·수정 시 반드시)

1. **토큰 상속**: 시스템 토큰 그대로. 필러 페이지면 틴트 4값(`--bg/--surface/--ink/--line`)만 교체.
2. **컨테이너·리듬 준수**: `--maxw`(1200/1180)·`--gut`·섹션 여백(120/74·108/70).
3. **섹션 문법 준수**: eyebrow→title→sub→근거→CTA. 리빌 `.r`/`.stagger`.
4. **컴포넌트 재사용**: §5 목록에서. 없으면 확정 파일에서 이식(신규 발명 금지).
5. **전환 수렴**: 최종 행동은 상담/진단/가이드/다운로드 하나로. 교육 상담 CTA는 원칙 6.
6. **B안 6대 원칙(§0.5) 우선 준수**: GNB 라우팅·로고 홈·톤 정합 그라데이션·인터랙션 고도화·공통 푸터·교육 상담 CTA.
7. **채널 분리 유지**: 홈=전사 신뢰, 상세=여정 근거.
8. **보이스 유지**: 선언형 평서문·대구·증거 기반.
9. **충돌 시**: 홈 확정본 > 코퍼레이트 > P1~P4. 단, §0.5 6대 원칙은 세부 규칙보다 우선.

> 이 문서와 어긋나는 화면은 "KEESS가 아니다." 일관성이 최우선이다. B안은 여기에 **§0.5 6대 원칙**을 반드시 얹는다.
