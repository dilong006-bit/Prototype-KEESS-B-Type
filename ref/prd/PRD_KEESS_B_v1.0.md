## I. 요구사항 분석 및 전략

### 문제 정의
- KEESS는 KG에듀원 HRD사업본부의 **B2B·B2G 인바운드 영업 채널**이다. 결제·가격이 없고, 제품 소개 → 상담/리드 인입 → 오프라인 영업 전환이 목적이다.
- 현재 B안 산출물은 **개별 정적 HTML 프로토타입 5종**으로 흩어져 있다. 이를 **하나의 일관된 사이트로 빌드·배포 가능한 상태**로 만들어야 한다.
- 검증된 디자인 언어(`Design.md`)와 B안 특화 기능(P3 정부지원 환급, P4 과정리스트 다운로드)이 **코드로 정확히 이식**되어야 한다.

### 솔루션 강점
- 확정 디자인 언어 100% 준수 → "하나의 사이트" 일관성.
- 필러별 정체성은 틴트 토큰으로만, 공통 컴포넌트 재사용 → 유지보수·A/B 재사용 용이.
- 모든 경로가 **상담/다운로드 리드로 수렴** → 인바운드 매출 기여.

## II. 비즈니스 모델 & 성공 메트릭

- **모델**: 무료 노출 인바운드 채널. 매출은 오프라인 영업 전환에서 발생. 사이트의 역할 = **정성 리드 최대화**.
- **핵심 가설**: 근거(진단·프레임워크·정부지원·규모)를 사이트 내에서 충분히 제시하면 외부 이탈이 줄고 상담 전환이 오른다.
- **KPI**
  - 1차(전환): 문의/상담 폼 제출 수(페이지별·관심영역별 태깅), 과정리스트 다운로드 수(리드), 정부지원 상담 수.
  - 2차(선행): 페이지 스크롤 깊이, 필러 페이지 도달률, 탐색기 사용률.

## III. 사용자 페르소나 & 여정

- **P-A 기업 HRD/교육 담당자(B2B)** — 연간 교육 계획·예산 담당. 관심: 체계·비용(환급)·운영. 진입: 검색/추천 → 필러 탐색 → 상담.
- **P-B 공공·기관 담당자(B2G)** — 위탁운영·법정 준수. 관심: 시스템·운영·인증. 진입: 홈 → P3.
- **P-C 중소기업 대표/관리자** — 소수 인원, 비용 민감. 관심: 정부지원 환급·법정필수. 진입: 정부지원/콘텐츠 → 상담.
- **공통 여정**: 진입 → 문제 인식(공감) → 근거 확인(체계·규모·환급) → **상담/다운로드**.

### 기능별 스펙 (요약 · ID·수용조건)

- **F1 공통 셸** — WHY: 상시 전환 경로·일관 프레임. HOW: 고정 Nav(스크롤 배경 전환), 모바일 오버레이 메뉴, 다크 Footer(전 페이지 공통), 페이지별 서브내비. **Design.md §0.5 6대 원칙 필수 반영**: ① GNB 각 항목=실제 페이지 라우팅(`/ax-ai`·`/leadership`·`/hrd`·`/content`, 정부지원=`/hrd#gov`), Next `<Link>`, 현재 페이지 `aria-current` — 무동작 금지. ② 좌측 상단 KEESS 로고=홈(`/`). ⑤ Footer는 전 페이지 공통(부정훈련 포함). ⑥ "교육 상담" CTA 최상 체감으로 강화(고대비 pill+아이콘·hover 상승·톤 정합 그라데이션 허용). **Done when**: 전 페이지 동일 셸, **GNB 클릭 시 각 페이지로 실제 이동**·현재 메뉴 `aria-current` 활성, **로고 클릭 시 홈 이동**, 공통 Footer 동일, 교육 상담 CTA 강화 확인, 모바일 토글 동작.

- **F4 P1** — 히어로·목표별 경로·End-to-End 5Step·AX Framework Lv1~5·직무 포지셔닝 맵·차별성·성과증거·라인업·CTA·모달. 구성 변경 금지(D 확정). **Done when**: 섹션·카피 일치.

- **F12 모션** — `.r`/`.stagger` IntersectionObserver(th .16 / .12), hover translateY, 등장 .7~.9s. reduced-motion 대응. **Done when**: 리빌·hover 동작, 모션 저감 대응.

## VI. UI/UX 설계 원칙 (요약, 상세 Design.md)
- 중립 베이스 + 필러 틴트(4값)만, 필러색은 포인트. 폰트 2종(Pretendard·Gowun Batang). 라운드/그림자/이징 토큰 고정.
- 섹션 문법: eyebrow→title→sub→근거→CTA. 선언형 카피, 숫자·기관 증거. 채널 분리(전사 신뢰=홈).
- **그라데이션 정책(Design.md §0.5-3)**: 확정 프로토타입이 원래 그라데이션을 쓰던 영역(히어로 배경·미디어 스크림·강조 CTA·데이터 시각화 바/노드)만 필러색·중립 파생 **톤 정합** 그라데이션 허용. 신규 다색 그라데이션 발명·대형 텍스트 배경 채색 금지.
- **인터랙션 고도화(§0.5-4)**: 구성·카피 불변 범위에서 hover/focus/전환/스크롤 스파이 등 체감 품질 향상 허용. **교육 상담 CTA(§0.5-6)는 가장 누르고 싶은 형태로 최우선 다듬음.**
- 모션: 스크롤 리빌·hover 상승, 1s 초과·바운스 금지, reduced-motion 대응.

## VII. 기술 아키텍처 개요 (상세는 TECHSPEC)
- **Next.js(App Router) + React + TypeScript + Tailwind CSS v3.4**(A-Type 빌드와 동일 스택). 정적 데이터, `next/font`.
- 폴더: `app/`(라우트), `components/common`·`components/sections`, `data/`, `lib/`, `public/`(폰트·이미지·다운로드 xlsx).
- 데이터: 페이지별 `data/*.ts`(카피=확정본), P4 과정 카탈로그(34종), P4 다운로드 통계, 폼 스키마(전송 없음).

## VIII. 오류 처리 및 엣지 케이스
