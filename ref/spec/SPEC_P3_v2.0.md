# 기술명세서 — P3 HRD 통합 솔루션 B안 v1.5 → v2.0

문서 목적: B안 P3에 **정부지원(사업주훈련) 환급** 섹션을 추가하고 문의 폼을 리드 전환에 맞게 확장. 기존 v1.5 디자인은 100% 보존(외과적 삽입). A안 미적용.
기준: `keess_P3_hrd_B_platform_v1.5.html`, `Design.md`, 공식 출처(고용노동부·한국산업인력공단 HRD4U).

## 1. 범위
- 신규 섹션 `#gov`(정부지원 환급) 1개 삽입 — 위치: `#trust`(운영 규모) 뒤 · `#inq`(도입 문의) 앞.
- `#inq` 관심영역 select 확장(정부지원 환급 옵션 + id 부여) 및 프리셀렉트 로직.
- 내비게이션 연동(gnav '정부지원' → `#gov` 스크롤, subnav 항목 추가).
- 크로스링크: `#gov` → P4(콘텐츠 솔루션) '환급 가능 과정 리스트'.
- 부수 품질보정: 문서 title·모달 카피의 P2 잔재 문구를 P3로 정정, 버전 표기 v2.0.

## 2. 목적/전략 (결정 반영)
- KEESS 존재 이유 = **간략 소개 → KG에듀원 상담 인입 → 추가 매출**. 따라서 정부 정보 백과가 아닌 **경량 전환 섹션**.
- 서사 앵커 = P3의 "HRD 운영 위탁"(B3-2). "복잡한 정부지원 절차를 KG에듀원이 대행" = 운영 위탁의 핵심 이점.
- 금액 표기 수위(확정): 이점은 노출하되 **단정 금지** — "우선지원기업 기준 최대 90%" + "규모·과정별 상이, 상담 시 산정" 캐비앗(기존 `예시` 라벨 관행 재사용).

## 3. `#gov` 섹션 명세
### 3-1. 구성(위→아래)
1. Eyebrow(정부지원 골드 배지): `GOVERNMENT SUPPORT · 정부지원 환급`
2. Title(선언·이점): "교육비, **정부지원으로 돌려받으세요**"
3. Lead(1문장): 고용보험 가입 사업주는 위탁훈련비 상당 부분 환급 — 인정신청~비용 정산까지 KG에듀원이 위탁 훈련기관으로 대행.
4. 증거 3카드(`.gov-grid`)
   - WHO · 지원 대상 = 고용보험 가입 사업주(규모 무관)
   - HOW MUCH · 환급 규모 = 훈련비 최대 90% 환급(우선지원기업 기준, 강조 카드 `.hi`)
   - WHAT WE DO · KG 대행 = 신청·실시신고·비용신청·정산 대행
5. 절차 4-step 스트립(`.gov-proc`, `--gov` border-top)
   - STEP1 인정신청(위탁 7일·자체 5일 전) → STEP2 훈련 실시·수료(집체 80% 출석 / 원격 진도 80%+평가) → STEP3 비용지원 신청(수료 14일 이내) → STEP4 지원금 지급(KG 대행)
6. 전환 밴드(`.gov-cta`, 다크): "우리 회사가 얼마나 지원받을까요?" + 보조 크로스링크(환급 가능 과정 리스트 → P4) + 1차 CTA "지원 가능 여부 상담 →"(→ `#inq`, 정부지원 프리셀렉트)
7. 법적 주석(`.gov-note`): 지원율·한도는 관계 법령·연도별 고시에 따름, 수치는 예시, 실제 상담 산정, 출처 표기.

### 3-2. 데이터(정적, 공식 출처 근거)
- 지원대상: 고용보험 가입 사업주 / 재직근로자·채용예정자·구직자.
- 지원율(우선지원기업, 위탁·원격 기준): 최대 90%(스마트훈련 95%) — 표기는 대표값 "최대 90%"만, 상세 표는 미노출(정보 과부하·법무 회피).
- 절차·기한: 인정신청 위탁 7일/자체 5일 전, 비용신청 수료 14일 이내, 지원금 훈련종료 3개월 내, 수료 집체 80%출석/원격 진도 80%+평가.
- 환급 과정 규모(P4 연계): 스마트러닝-환급 과정 495개+.

### 3-3. 스타일 토큰(신규 컬러 없음)
- 베이스: P3 바이올렛 틴트 유지(`--bg/--surface/--ink/--line`, `--mg/--mg2/--mg3/--acc`).
- 포인트: `--gov:#F4B83A`(Design.md 확정 토큰, `:root`에 추가) — eyebrow 배지·강조카드 보더·step 넘버·숫자에만 사용(크로스 배지 원칙).
- 1차 CTA는 기존 `.btn-mg`(마젠타 그라디언트) 재사용 = 사이트 CTA 일관성.
- 리빌: 기존 `.r`(IntersectionObserver th .12) 재사용.

## 4. `#inq` 폼 변경
- select에 `id="fArea"` 부여, 옵션 정리 + 정부지원 추가:
  `HRD 통합 솔루션(P3)` · `AX·AI 전환(P1)` · `콘텐츠 솔루션(P4)` · `정부지원 환급` (기존 중복 'HRD 통합(P3)' 제거).
- 신규 함수 `goToInq(area)`: `#inq`로 스크롤 + 관심영역 프리셀렉트(값/텍스트 매칭). `#gov` CTA가 `goToInq('정부지원 환급')` 호출 → 리드 소스 태깅.

## 5. 내비게이션 연동
- gnav '정부지원' `<a>`에 `onclick="goTo('gov')"`.
- subnav에 `<a data-t="gov">정부지원</a>`를 '운영 규모'와 '도입문의' 사이에 추가(섹션 순서 일치, 스크롤 스파이 자동 적용).

## 6. 반응형·접근성
- `.gov-grid` 3→1열(≤820), `.gov-proc` 7컬럼(스텝+화살표)→1열(≤820, 화살표 숨김).
- 전환 밴드 flex-wrap로 모바일 세로 정렬.
- 색 대비: 골드 텍스트는 배지/보더 등 큰 요소·굵은 숫자에만(본문 가독 유지).
- 링크·버튼 `:focus-visible` 기존 규칙 상속, 크로스링크는 실제 `<a>`.
- `prefers-reduced-motion`은 기존 전역 정책 상속.

## 7. 수용 기준(Definition of Done)
- 기존 8개 섹션(hero/arch/studio/demo/ai/modules/trust/inq)·데모·탭·휠 등 인터랙션 무손상.
- `#gov`가 trust와 inq 사이에 렌더, 리빌 정상.
- gov CTA 클릭 → inq 스크롤 + 관심영역 '정부지원 환급' 자동 선택.
- gnav/subnav '정부지원' 클릭 → `#gov` 스크롤, 스크롤 스파이 활성.
- 신규 컬러 토큰은 `--gov` 1개(Design.md 기존값)뿐, 그 외 신규 값 0.
- 데스크톱/태블릿/모바일 레이아웃 정상.
