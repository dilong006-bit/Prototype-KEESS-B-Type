# P4 콘텐츠 솔루션 B안 — 업무 전략 + 기술명세서 (v1.3 → v2.0)

기준: `keess_P4_content_solution_B_v1.3.html`, `Design.md`, `KG에듀원_과정리스트.xlsx`(실측). 원칙: 기존 v1.3 디자인 100% 보존(외과적 삽입), A안 미적용, 신규 컬러·컴포넌트 발명 금지.

## PART 1. 업무 전략

### 1-1. 목적
- 현재 라이브 KEESS는 **우측 플로팅 '과정리스트 다운로드'**로 엑셀을 제공. → B안에서는 **플로팅 노출을 쓰지 않고 P4 자연 흐름 안에서** 다운로드를 제공.
- KEESS 존재 이유 = 소개 → **상담/리드 인입 → KG에듀원 추가 매출**. 따라서 다운로드를 단순 파일 제공이 아닌 **리드 자석 + 라이트 게이트**로 설계.

### 1-2. 자산 실측(엑셀)
- 8시트, 약 **8,000+ 과정**: 직무(공통) 4,219(스마트러닝-환급 495) · 외국어 3,332 · 전화외국어 ~820 · 법정필수 25 · 이달 신규 116 · 추천 · 정기개강일. **매월 갱신**(2026-08 개강 기준).
- 시사점: "규모 + 최신성 + 환급"을 증명하는 리드 자석. 특히 **환급 495개**가 P3 정부지원과 직접 연결.

### 1-3. 배치 전략(자연 흐름)
- P4_B 흐름: hero → axisnav → #ax1 직무 → #ax2 어학 → #ax3 IT·자격 → #ax4 비즈·리더십 → #ax5 법정 → #ax6 제작·파트너 → final(문의).
- **신규 `#download` 섹션을 #ax6 뒤 · final(문의) 앞**에 배치. "방금 6체계를 봤으니, 전체를 한 파일로" 흐름. 플로팅 다운로드는 신설하지 않음(요구 반영).
- 기존 플로팅 FAB('대표 과정 찾기' 탐색기)는 다운로드가 아니라 확정 기능이므로 유지.

### 1-4. 다운로드 UX(라이트 게이트 · 확정)
- 다운로드 CTA 클릭 → **최소 정보 게이트 모달**(담당자·회사·이메일) → 제출 시 **엑셀 자동 다운로드 + 성공 상태**(수동 다운로드 링크 폴백). 무료 감성 유지 + 리드 확보.
- 게이트 모달은 기존 문의 모달(`form-modal`) 패턴 재사용.

### 1-5. 크로스 연결(P3 ↔ P4)
- `#download` 블록에 "환급 가능 과정 495개 포함 · 정부지원 환급 안내 → P3" 보조 링크(→ `keess_P3_hrd_B_platform_v2.0.html#gov`).
- P3의 gov 섹션에는 이미 "환급 과정 리스트 → P4" 링크가 있음(양방향 루프 완성).

### 1-6. 이미지·아이콘 소스
- 실사: Unsplash(교체 슬롯 `imgslot` 관행 유지). 아이콘: 인라인 SVG(stroke 1.8) 유지 — 필요 시 Flaticon/Iconify에서 동형 라인 아이콘으로 교체 가능(신규 색 없이 currentColor 상속).

## PART 2. 기술명세서

### 2-1. 범위
- 신규 `#download` 섹션 1개(#ax6 뒤·final 앞).
- 신규 다운로드 게이트 모달 `#ov-download`.
- 다운로드 자산 `KG에듀원_과정리스트.xlsx`(폴더 동봉, 상대경로 링크).
- JS: `data-download` 트리거 · openDownload · 폼 검증 · 자동 다운로드(triggerDL).
- 부수: 푸터 버전 v2.0 표기.

### 2-2. `#download` 섹션 구조
- 2단 카드(`.dl`, 1.05fr / .95fr, ≤820 1열):
  - 좌: eyebrow(DOWNLOAD · 전체 과정 리스트) → H2("필요한 과정을 한눈에, 전체 리스트로") → lead → **크로스링크(정부지원, 바이올렛)** → CTA(`data-download`, "과정 리스트 다운로드 (Excel)") + 메타(.xlsx·매월 갱신·정기개강일).
  - 우: 파일 카드(`.dl-file`) — XLS 배지 + 파일명/기준일 + **통계 6칸**(직무 4,219 / 어학 3,332 / 전화외국어 820 / 법정 25 / 신규 116 / 환급 495+[violet]) + 갱신 주석.

### 2-3. 데이터(정적, 엑셀 실측)
- 직무 4,219 · 어학 3,332 · 전화외국어 820 · 법정 25 · 신규 116 · 환급 495+ · 기준 2026-08 개강 · 매월 갱신.
- 총량 표기는 hero 기존값 "8,426"과 정합(그대로 유지, 리스트 총량은 "8,000여").

### 2-4. 게이트 모달 `#ov-download`
- `form-modal` 재사용. 필드: 담당자명*·회사/기관*·이메일*(ids `df-name/df-org/df-mail`), 안내 ctx.
- submit(`#df-submit`): 검증(공란·이메일 형식) → 실패 시 border 강조 → 성공 시 폼 숨김 + `#d-ok`(성공) 표시 + `triggerDL()` 자동 다운로드. `#d-ok`에 수동 다운로드 링크(`<a download>`) 폴백.
- 개폐: 기존 `.ov`/`data-close`/배경클릭/ESC 로직 재사용(포커스 관리 포함).

### 2-5. JS 배선
- `data-download` 전용 click 리스너 추가(기존 위임과 분리, 비침습).
- `triggerDL()`: 동적 `<a href="./KG에듀원_과정리스트.xlsx" download>` 클릭.

### 2-6. 토큰·컴포넌트(신규 컬러 0)
- P4 오렌지 토큰(`--or/--or2/--orsoft/--orInk`) + 크로스링크만 P3 바이올렛(`--rp/--rpsoft`) 재사용. XLS 배지는 스프레드시트 관습색(녹색 #1E6B41, 로컬 스코프).
- `.btn-or`, `.eyebrow`, `.lead`, `.field`, `.okmsg`, `.ov`, `.form-modal`, `.shadow` 전부 기존 클래스.

### 2-7. 반응형·접근성
- `.dl` 2→1열(≤820), `.dl-stats` 3→2열(≤420). 모달 `role="dialog"`·`aria-modal`·ESC·포커스 이동(기존). CTA/링크 `:focus-visible` 상속.
- `prefers-reduced-motion` 전역 정책 상속.

### 2-8. 수용 기준(DoD)
- 기존 6축·탐색기·문의·스크롤스파이 무손상.
- `#download`가 #ax6와 final 사이 렌더.
- 다운로드 CTA → 게이트 모달 → 제출 → 엑셀 다운로드 + 성공 표시.
- 크로스링크가 P3 v2.0 #gov로 연결.
- 신규 컬러 토큰 0(로컬 XLS 녹색 제외), 데스크톱/태블릿/모바일 정상.

### 2-9. 범위 외
- 플로팅 다운로드 신설(요구상 배제). 실제 메일 발송/리드 저장(프로토타입은 상태 UI + 로컬 다운로드).
- 엑셀 내용 편집(사용자 롤 아님).
